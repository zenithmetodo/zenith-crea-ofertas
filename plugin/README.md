## 🚀 Instalación · 1 línea

### Para clientes nuevos · Mac/Linux

```bash
git clone https://github.com/zenithmetodo/zenith-crea-ofertas.git ~/.claude/skills/zenith-crea-ofertas && bash ~/.claude/skills/zenith-crea-ofertas/plugin/install.sh
```

### Para clientes nuevos · Windows (PowerShell admin)

```powershell
git clone https://github.com/zenithmetodo/zenith-crea-ofertas.git "$env:USERPROFILE\.claude\skills\zenith-crea-ofertas"; & "$env:USERPROFILE\.claude\skills\zenith-crea-ofertas\plugin\install.ps1"
```

### Si ya tienes Claude Code abierto

```
/setup-crea-ofertas
```

✅ Python 3 · pip · ffmpeg · Whisper (transcripción vídeos) · jq · BeautifulSoup (scraping research)

## 🏔️ ¿Qué hace esta skill?

Construye, **palabra por palabra**, una oferta completa de alto valor percibido bajo el **Método Zenith™**:

### Bloque AVATAR + INVESTIGACIÓN (v2.0.0 · nuevo)
- **Avatar profundo** (11 componentes · psicólogo experto · día en la vida + archetype name + lenguaje literal)
- **Mapa de empatía** (siente/piensa/ve/hace · antes vs después)
- **Research Reddit** (30-50 frases textuales del avatar · subreddits mapeados · upvotes reales)
- **Research YouTube** (top creadores · vídeos virales · comentarios virales · mecanismos ya nombrados)
- **Competidor Spy** (Biblioteca de Anuncios Meta · gap competitivo identificado)
- **Deal-makers + Deal-breakers** (10 cosas que SÍ hacen comprar + 10 que LE HACEN HUIR)

### Bloque OFERTA (heredado de v1.0 + v1.1)
- **Punto A → Punto B** del cliente (antes vs después · 5 dimensiones)
- **One Belief Statement** en **4 variantes** (identidad / anti-identidad / nueva oportunidad / combo)
- **Mecanismo desdoblado** (problema = villano + solución = cómo deseable + nombre chicle 2-3 palabras + super estructura cultural Stage 5)
- **Características → Beneficios → Deseos profundos** (Maslow + Robbins 6 needs + Bauer 8 fears + Jung arquetipos)
- **Promesa ganadora** (fórmula 7 componentes del autor · test del ascensor)
- **Garantía impresionante** (11 tipos · nombre atractivo) + **Risk Reversal Stack** (3-7 capas Hormozi)
- **Módulos + Clases** con la fórmula "carta de ventas invisible"
- **6 categorías de bonus** + transformación a irresistibles + estrategia Charles Cesar
- **Valor percibido máximo** (fórmula Bencivenga + Stack Slide Brunson + Anclaje de precio)
- **Upsells / Downsells / Order bumps** (AOV +30-100%)
- **Headline-master** (Halbert + Caples + Ogilvy · 27 fórmulas)
- **Fascinations** (25-40 bullets hipnóticos)
- **Autoridad builder** (Cialdini · construir credibilidad desde cero)

### Bloque ÁNGULOS (v2.0.0 · nuevo)
- **5-10 ángulos** con definición exacta del autor: *"razón distinta de por qué me comprarían"*
- 3 ingredientes obligatorios: **tipo concreto de persona + creencia específica + reconocimiento del problema con solución**
- 3-5 **sub-ángulos** por cada ángulo
- Matriz Schwartz × Stage × Lead → ángulo correcto

### Bloque CONSCIENCIA + CIERRE
- **Schwartz** (5 niveles + 5 stages) auditados pieza por pieza
- **Great Leads Masterson + Forde** (6 tipos · elegir el correcto)
- **Oferta completa ensamblada** (Hook-Story-Offer + Stack Slide + 17 secciones)
- **Auditoría 200+ ítems** con score · roadmap rojo/ámbar/verde · veredicto luz-verde/ámbar/roja

**Todo entregado en carpetas estéticas Zenith con HTML imprimibles a PDF + visualizables en Drive.**

## ⚡ Optimización de tokens (estrategia model-by-task)

Cada agente declara su `model` según la complejidad de su tarea:

| Model | Cuándo | # Agentes |
|---|---|---|
| **🧠 Opus** | Decisiones estratégicas · razonamiento creativo profundo · auditoría maestra | 12 |
| **🧠 Sonnet** | Generativo con razonamiento · investigación · transformación inteligente | 23 |
| **⚡ Haiku** | Extracción simple · procesamiento rutinario · conversión de formato | 6 |

### Reglas de optimización aplicadas

1. **Knowledge INTERNALIZADO** en cada agente · NO releen archivos `knowledge/` en ejecución
2. **Solo leen brief + output anterior** · NO exploran carpetas
3. **Output bounded** · cada agente declara tamaño máximo
4. **El padre orquesta · los hijos son cortos**
5. **Bash solo para ACCIONES** · no para cat/ls
6. **Reuso de outputs JSON** · NO regeneración
